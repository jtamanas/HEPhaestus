# hep-ph-agents Design System

**Date**: 2026-04-16
**Status**: Approved
**Scope**: Visual identity system + output design system for all hep-ph-agents plugin outputs

## Overview

A design system for high-energy physics phenomenology plots and diagrams. The goal is recognition through consistency — plots generated by hep-ph-agents skills should be identifiable across papers not by a watermark but by a shared visual discipline that no other tool produces.

No existing HEP plotting guide applies Tufte-informed dataviz principles to physics figures. The ATLAS/CMS style sheets are branding exercises. This system fills the gap: a craft-first design language for phenomenology.

## Philosophy

**One-liner**: *Every visual difference encodes a physics difference. Subtract everything else.*

### Five principles, in priority order

1. **Visual dimensions = model-space dimensions.** Hue maps to model identity. Lightness maps to parameter variation. Line style maps to approximation level or subordination. If two things look different on the plot, they must mean something different in the physics. No decorative variation, ever.

2. **Direct over indirect.** Labels live on the data. Annotations sit next to the thing they describe. No legend boxes. No cross-referencing. A reader understands a curve without moving their eyes elsewhere on the figure.

3. **Subtract, don't add.** No gridlines. No bounding boxes. No background fills. Start with everything small, thin, and gray — then selectively promote the elements that carry meaning. You must justify every piece of visual emphasis; the default is invisible.

4. **Build for print reality.** Figures are built at the exact target column width (85mm single-column, 170mm double). Never scale in LaTeX. Font sizes target ~8-9pt at final print dimensions. Line weights stay between 0.25pt and 1.5pt at print. What you export is what the reader sees.

5. **Semantic color only.** Color is never decorative. Each hue in the palette has a fixed semantic role. De-emphasis uses desaturation of the accent hue, not a new hue. Grays are warm-tinted (pushed slightly toward the dominant accent), never pure neutral.

### Influences

- Edward Tufte — data-ink ratio, chartjunk elimination, small multiples
- Claus Wilke, *Fundamentals of Data Visualization* — balancing data and context, redundant coding
- Lisa Charlotte Muth / Datawrapper blog — color restraint, direct labeling, annotation hierarchy
- Pascal Michaillat — minimalist scientific figure templates with exact numerical specs
- Allan Chain — matplotlib rcParams calibrated for physics papers

## Color System

### Two palette contexts, one shared grammar

#### Palette A — Cool Analytic (theory-vs-data plots)

| Role | Color | Hex | When |
|------|-------|-----|------|
| Theory prediction | Blue | `#0284c7` | Calculated curves, NLO/NNLO predictions |
| Experimental data | Burnt orange | `#ea580c` | Measured points, error bars |
| Exclusion/secondary | Violet | `#7c3aed` | Uncertainty bands, secondary exclusion |
| Ink | Near-black | `#1a1a1a` | Axes, ticks, primary text |
| De-emphasis gray | Warm gray | `#9ca3af` | Subsidiary labels, annotations |

#### Palette B — Slate Precision (exclusion contours)

| Role | Color | Hex | When |
|------|-------|-----|------|
| Theory prediction | Steel blue | `#0369a1` | Predicted limits, theory curves |
| Observed data | Black | `#1a1a1a` | Observed limit (highest-contrast element) |
| Excluded region | Muted red | `#dc2626` | Ruled-out parameter space (low-opacity fill) |
| Ink | Slate | `#334155` | Axes, ticks, labels |
| De-emphasis gray | Slate gray | `#94a3b8` | Subsidiary structure |

#### Palette C — Ink-only (Feynman diagrams)

| Role | Color | Hex | When |
|------|-------|-----|------|
| Fermion/scalar lines | Near-black | `#1a1a1a` | External legs, propagators |
| Boson of interest | Context accent | `#0284c7` or `#0369a1` | The propagator or vertex being discussed |
| Labels | Dark gray | `#555555` | Particle labels in italic CM Serif |

### Multi-curve escalation rules

When multiple theory curves share a plot:

1. **Same model, different parameters** → same hue, vary lightness. A 4-step ladder within each hue family:
   - Blue: `#0c4a6e` → `#0369a1` → `#0ea5e9` → `#7dd3fc`
   - Violet: `#4c1d95` → `#7c3aed` → `#a78bfa` → `#c4b5fd`
   - Teal: `#134e4a` → `#0f766e` → `#14b8a6` → `#5eead4`

2. **Same model, subordinate variant** → same hue + same lightness, dashed. Solid = primary prediction. Dashed = limiting case, different approximation order, or secondary parameter choice.

3. **Different models** → different hue. Second model gets violet (`#7c3aed`), third gets teal (`#0f766e`). Each can have its own solid/dashed pair.

4. **Hard cap: 7+ curves → small multiples.** If the escalation ladder is exhausted, the answer is never "more colors." It's separate panels with a shared axis.

### Brazil bands (special case)

The green/yellow expected-limit convention is too established to replace, but we mute it: desaturated fills at alpha 0.2-0.3, observed limit as solid black (highest contrast), expected limit as thin dashed, direct labels instead of legend entries.

### Fill opacity rules

- Exclusion region fills: alpha 0.06-0.10 (barely tinted, boundary line does the work)
- Uncertainty bands (1σ): alpha 0.12-0.15
- Uncertainty bands (2σ): alpha 0.06-0.08
- Brazil bands (±1σ green): alpha 0.25, desaturated
- Brazil bands (±2σ yellow): alpha 0.20, desaturated
- When more than 2 bands overlap: switch to small multiples rather than stacking transparency

## Typography

**One font family: Computer Modern Serif.** Every label, annotation, and axis title looks like it was typeset in the same system as the paper's equations.

| Element | Size (at print) | Weight | Style | Color |
|---------|-----------------|--------|-------|-------|
| Axis labels (σ, m_χ, etc.) | 9pt | Regular | Italic | Ink |
| Tick labels | 8pt | Regular | Roman | Ink |
| Direct annotations ("NLO", "LHC") | 8pt | Regular | Italic for theory, Roman for data | Matches curve color |
| Subsidiary text ("excluded", "95% CL") | 7pt | Regular | Roman | De-emphasis gray, 60% opacity |
| Units in brackets ([GeV], [pb]) | 8pt | Regular | Roman | Ink |

### Rules

- Exactly two text hierarchy levels. Primary (9pt ink) and secondary (7-8pt gray). Never a third.
- Left-align all annotations. Never center.
- LaTeX rendering for all math — no matplotlib's built-in math font.
- Particle symbols always italic. Units always roman.

## Line Weight Hierarchy

Data dominates, structure recedes. All values at final print size:

| Element | Weight | Purpose |
|---------|--------|---------|
| Primary data/theory curves | 1.0-1.2pt | What the reader is here to see |
| Uncertainty band boundaries | 0.5pt, dashed | Subordinate to the central value |
| Axis lines | 0.5pt | Structural, not informational |
| Tick marks | 0.5pt wide, 3pt long (major), 1.5pt long (minor) | Orientation only |
| Error bars | 0.7pt | Visible but thinner than the curve they annotate |
| Exclusion region boundary | 1.2pt | As prominent as data — this is a result |

**No gridlines.** If a reader needs to trace a value to an axis, the tick marks are sufficient. If they can't, the plot needs fewer curves, not more grid.

## Layout & Composition

### Figure dimensions

Build at exact target. Never scale in LaTeX.

| Context | Width | Aspect | Use |
|---------|-------|--------|-----|
| Single-column (PRL, PRD, JHEP) | 85mm | 4:3 | Default for most plots |
| Double-column / full-width | 170mm | 16:9 | Small multiples, wide exclusion scans |
| Slides / talks | 170mm | 16:9 | Same as double-column — one asset serves both |

### Axis treatment

- No bounding box. Only left axis and bottom axis. The data defines the upper and right boundaries.
- Ticks face inward.
- Axis lines meet at the origin corner but don't extend beyond the data range.
- Log-scale tick marks: major ticks at each power of 10, minor ticks at 2-9. No labels on minor ticks.

### Whitespace

- Margin between axis labels and tick labels: 4pt minimum.
- Margin between plot area and direct annotations: 2pt minimum from any curve.
- Padding around the entire figure: tight bounding box (`bbox_inches='tight'`, `pad_inches=0.02`).
- Between small-multiple panels: 8mm horizontal, 6mm vertical.

### Annotation placement

- Direct labels sit at the end of the curve — right side, vertically aligned with the curve's final value. If curves converge, stagger labels vertically with 2pt minimum spacing.
- Region labels ("excluded", "95% CL") sit inside the region, lower-right corner, in de-emphasis gray at 60% opacity.
- Annotations go inside the plot area, never in a separate text block.

### Theory-vs-data specific layout

- Ratio panels share the x-axis with the main plot. Height = 30% of main panel. Separated by a thin 0.5pt line, no gap. Y-axis labeled "Data/Theory" or "Ratio" with a horizontal reference line at 1.0 in de-emphasis gray.

### Feynman diagram specific layout

- Square aspect ratio.
- Generous padding — external legs extend to at least 30% of the diagram width before the vertex.
- Particle labels: 2pt offset from the line, italic CM Serif, placed at the midpoint of external legs.
- Propagator labels: centered above/below the propagator, in the accent color.
- Vertex dots: 2pt radius, filled black. Only at interaction vertices, never at line endpoints.

## Implementation Architecture

### Three layers

**1. Design Reference Document** (`docs/design-system.md`)
The source of truth. Contains every principle, rule, and value. Any skill references this document for judgment calls.

**2. matplotlib Style Sheets** (`styles/`)
- `hep-ph-agents-analytic.mplstyle` — Cool Analytic palette (theory-vs-data)
- `hep-ph-agents-slate.mplstyle` — Slate Precision palette (exclusion contours)

Encodes every rule that matplotlib can express declaratively: line weights, font sizes, tick parameters, axis styling, figure dimensions, color cycle.

**3. TikZ Style Package** (`styles/hep-ph-agents-tikz.sty`)
LaTeX style file for TikZ-Feynman diagrams. Defines line weights, vertex styles, label positioning, accent color, and propagator rendering.

### Skill integration

Each skill's SKILL.md gets a **Style** section that:
- References the design reference doc
- Specifies which palette context to use by default
- Notes any plot-type-specific rules (e.g. Brazil band handling, ratio panel layout)

Mapping:
- `hep-plot` → Cool Analytic, multi-curve escalation rules
- `exclusion-contour` → Slate Precision, Brazil band rules
- `draw-feynman` → Ink-only, TikZ style package
- `feynman-tikz` → Ink-only, TikZ style package
- `theory-data-comparison` → Cool Analytic, ratio panel layout rules

### Python helper module (`hep_ph_style.py`)

Convenience functions for things `.mplstyle` can't express:
- `apply_direct_labels(ax, curves)` — endpoint labels with collision avoidance
- `add_uncertainty_band(ax, x, y, yerr, ...)` — correct alpha and boundary styling
- `escalate_colors(n_models, palette)` — returns colors + line styles per escalation ladder
- `set_hep_context(context='analytic'|'slate')` — loads the right `.mplstyle` and color cycle

This module is a convenience, not a requirement. Every rule is also in the reference doc for use with ROOT, pgfplots, or any tool.

### Project identity

The project identity inherits from the plot system. README uses Cool Analytic blue (`#0284c7`) as its only accent. No logo — the plots are the identity.
