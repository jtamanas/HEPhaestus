# Fix Evidence: sd-A-006 — Unicode glyph rendering (χ, Ω) in cmr10 font

**File**: `plugins/hep-ph-demo/skills/singlet-doublet/SKILL.md`
**Location**: Step 4d plotting code block, `ax.text()` call

## Old text
```python
ax.text(0.02, 0.98,
        f"Singlet-Doublet χ₁ · m={m_chi1:.1f} GeV\n"
        f"Ω h² = {results['Omegah2']:.3f} (Planck 0.120)\n"
        "arXiv:2506.19062 §II",
        transform=ax.transAxes, va="top", ha="left",
        fontsize=7, color=palette["deemph"])
```

## New text
```python
ax.text(0.02, 0.98,
        r"Singlet-Doublet $\chi_1$" + f", m={m_chi1:.1f} GeV\n"
        r"$\Omega h^2 = $" + f"{results['Omegah2']:.3f}" + " (Planck 0.120)\n"
        "arXiv:2506.19062 Sec. II",
        transform=ax.transAxes, va="top", ha="left",
        fontsize=7, color=palette["deemph"])
```

## Rationale
PT-A issue evidence: "UserWarning: Glyph 967 (GREEK SMALL LETTER CHI) missing
from font(s) cmr10". When `text.usetex=False` (LaTeX unavailable, mathtext
fallback active), Unicode characters U+03C7 (χ), U+2081 (₁), U+03A9 (Ω), and
U+00B7 (·) are not in the cmr10 font. Switching to LaTeX math-mode strings
(`$\chi_1$`, `$\Omega h^2$`) works correctly in both usetex=True (via LaTeX)
and usetex=False (via matplotlib mathtext with fontset=cm). The `§` section
sign was also replaced with "Sec." for ASCII safety in non-usetex rendering.
